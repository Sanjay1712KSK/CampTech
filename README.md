# 🚀 GigShield — Intelligent Insurance for Gig Workers

## 🏗 Core Platform

GigShield is built as one connected insurance system, not a collection of isolated demos.

- `User onboarding`: role selection, signup, OTP verification, account confirmation, DigiLocker-style verification, biometric support, and secure login
- `Gig account integration`: connects worker identity with delivery income history and earning baselines
- `Environment intelligence`: uses weather, AQI, and traffic signals to understand disruption in real time
- `Risk engine`: converts hyperlocal conditions into explainable risk and expected income loss
- `Premium generation`: prices weekly protection directly from the live risk profile
- `Policy creation`: activates coverage after payment and maintains policy state
- `Claim engine`: uses parametric disruption logic to auto-evaluate claim eligibility
- `Payout system`: supports claim approval flow, bank linking, and payout tracking
- `Blockchain adapter`: logs policy, claim, and payout records through a mock-first trust layer

## 🔄 How the System Works

GigShield follows a simple, connected pipeline:

`User -> Gig Data -> Environment -> Risk -> Premium -> Policy -> Disruption -> Claim -> Fraud -> Payout -> Blockchain`

- worker data establishes the earning baseline
- environment signals measure real disruption
- risk drives premium and coverage logic
- disruption plus income loss can trigger claims automatically
- fraud checks protect the insurer before payout
- payout and records are written into the trust layer

## 🧪 How to Run Locally

### Backend

```bash
git clone https://github.com/Sanjay1712KSK/GuideWire.git
cd GuideWire/backend
pip install -r requirements.txt
uvicorn main:app --reload
```

Backend runs at `http://127.0.0.1:8000`.

### Frontend

```bash
cd GuideWire
flutter pub get
flutter run --dart-define=API_BASE_URL=http://YOUR_LOCAL_IP:8000
```

### APK

```bash
flutter build apk --release
```

## 🏛 Tech Stack

- `Frontend`: Flutter, Riverpod
- `Backend`: FastAPI, SQLAlchemy
- `Database`: SQLite with PostgreSQL-ready deployment support
- `Payments`: Razorpay test integration
- `Verification`: OTP flows, DigiLocker-style KYC
- `Signals`: Open-Meteo, OpenWeather AQI, OpenRouteService
- `Security`: JWT, bcrypt, device binding, anomaly checks
- `Trust layer`: blockchain adapter with mock-first logging
